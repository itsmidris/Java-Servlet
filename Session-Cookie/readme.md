# Session & Cookies in Java Servlets

## Overview

HTTP is a **stateless protocol**, which means the server does not remember previous requests from a client.

Example:

1. User logs in.
2. Server validates credentials.
3. Response is sent.
4. User opens Profile page.
5. A new request is generated.

The server needs a way to identify that both requests belong to the same user.

This is achieved using **Cookies** and **Sessions**.

---

# Cookies

A Cookie is a small piece of data stored in the browser.

## Flow

```text
Browser ----> Login Request ----> Server

Server creates Cookie
Cookie(username=admin)

Server ----> Response + Cookie ----> Browser

Browser stores Cookie
```

For future requests:

```text
Browser ----> Request + Cookie ----> Server
```

The server reads the cookie and identifies the user.

---

## Create Cookie

```java
Cookie cookie = new Cookie("username", username);
response.addCookie(cookie);
```

---

## Read Cookie

```java
Cookie[] cookies = request.getCookies();
```

---

## Delete Cookie

```java
cookie.setMaxAge(0);
response.addCookie(cookie);
```

---

## Set Expiry Time

```java
cookie.setMaxAge(60 * 60);
```

Cookie expires after 1 hour.

---

## Limitations of Cookies

* Stored in browser
* User can delete them
* Limited storage
* Less secure

Never store:

* Passwords
* OTPs
* Bank details

---

# Sessions

A Session stores data on the server side.

Sessions are more secure than cookies because sensitive information remains on the server.

---

## Create Session

```java
HttpSession session = request.getSession();
```

---

## Store Data

```java
session.setAttribute("username", username);
```

---

## Retrieve Data

```java
String username =
    (String) session.getAttribute("username");
```

---

## Remove Attribute

```java
session.removeAttribute("username");
```

---

## Destroy Session

```java
session.invalidate();
```

Used during logout.

---

# What is JSESSIONID?

When a session is created:

```java
HttpSession session = request.getSession();
```

Tomcat generates a unique session identifier.

Example:

```text
JSESSIONID=ABC123XYZ
```

This ID is stored in a cookie.

For every request:

```text
Browser ---> JSESSIONID ---> Server
```

Tomcat uses the Session ID to locate the correct session object.

---

# Session vs Cookie

| Feature         | Cookie      | Session           |
| --------------- | ----------- | ----------------- |
| Stored In       | Browser     | Server            |
| Security        | Less Secure | More Secure       |
| Capacity        | Small       | Large             |
| User Can Delete | Yes         | No                |
| Used For        | Preferences | Login Information |

# Project Flow

## Step 1

Open:

```text
http://localhost:8080/Session-Cookie/login.html
```

---

## Step 2

Enter:

```text
Username : admin
Password : 1234
```

---

## Step 3

Form submits:

```html
<form action="login" method="post">
```

Request:

```http
POST /login
```

---

## Step 4

LoginServlet executes:

```java
String username =
        request.getParameter("username");

String password =
        request.getParameter("password");
```

---

## Step 5

Session created:

```java
HttpSession session =
        request.getSession();

session.setAttribute("username", username);
```

---

## Step 6

Cookie created:

```java
Cookie cookie =
        new Cookie("username", username);

response.addCookie(cookie);
```

---

## Step 7

WelcomeServlet reads Cookie:

```java
Cookie[] cookies =
        request.getCookies();
```

Output:

```text
Welcome admin
```

---

## Step 8

ProfileServlet reads Session:

```java
String username =
        (String) session.getAttribute("username");
```

Output:

```text
Profile Page
User : admin
```

# Annotation-Based Servlet Mapping

Instead of using web.xml mappings:

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
}
```

```java
@WebServlet("/welcome")
public class WelcomeServlet extends HttpServlet {
}
```

```java
@WebServlet("/profile")
public class ProfileServlet extends HttpServlet {
}
```

Advantages:

* Less XML configuration
* Easier maintenance
* Preferred in modern Servlet applications

# Interview Questions

### Why is Session more secure than Cookie?

Because Session data is stored on the server, while Cookie data is stored in the browser.

---

### What is JSESSIONID?

A unique session identifier generated by Tomcat to track a user's session.

---

### How do you destroy a session?

```java
session.invalidate();
```

---

### Can Session work without Cookies?

Yes.

Using URL Rewriting:

```text
/profile;jsessionid=ABC123XYZ
```

---

# Key Takeaways

* HTTP is stateless.
* Cookies store data in the browser.
* Sessions store data on the server.
* JSESSIONID connects browser requests to server sessions.
* Use Cookies for lightweight client-side data.
* Use Sessions for authentication and user-specific data.
* Modern Servlet projects prefer `@WebServlet` annotations over XML mappings.
